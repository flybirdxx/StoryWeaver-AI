
1. 视觉与风格增强 (Visual & Styling)
引入“深色模式” (Dark Mode) [高优先级]

原因：创作者（尤其是影视和绘图领域）习惯在暗色环境下工作，以减少屏幕眩光并更准确地感知色彩。目前的 bg-stone-50 界面虽然干净，但在长时间工作中可能会刺眼。

建议：利用 Tailwind 的 dark: 修饰符。背景可使用 bg-stone-900，卡片使用 bg-stone-800，文字反白。

强化视觉层级 (Visual Hierarchy)

现状：目前的概览页（Dashboard）文字较多，KPI 卡片和文本块的视觉权重比较平均。

建议：

将关键数字（如“Token 消耗”、“分镜数量”）放大并加粗。

为“AI 流水线”中的不同阶段（分析、导演、视觉）使用更明显的颜色编码（例如：分析=蓝色，导演=紫色，视觉=橙色），并在整个应用中保持这种颜色语义的一致性。

2. 交互体验优化 (Interaction & UX)
AI 处理的“感知等待” (Perceived Latency)

痛点：Gemini 3 Pro 生成图像和分析剧本都需要时间（几秒到几十秒）。单纯的“加载中”转圈会让用户感到焦虑。

建议：设计一个**“透明化思维链”**的加载状态。

不要只显示 Spinner。

显示实时步骤文本："正在解析场景..." -> "识别角色 Akira..." -> "构建光影提示词..." -> "正在渲染..."。这不仅缓解等待焦虑，还能让用户觉得 AI 很智能。

分镜画布的“操作感”

现状：目前的漫画布局演示是通过按钮切换 CSS class。

建议：增加拖拽手柄 (Drag Handles) 的视觉暗示。即使原型阶段不能真的拖拽，也要在分镜格子的边缘增加鼠标悬停样式（cursor-col-resize 或 cursor-row-resize），暗示未来这是可调节的。

3. 移动端适配 (Mobile Experience)
导航优化

现状：移动端使用了一个 <select> 下拉框进行导航。

建议：这是 Web 1.0 的做法。建议改为底部导航栏 (Bottom Tab Bar) 或 汉堡菜单 + 侧滑抽屉 (Hamburger + Drawer)。对于这种功能复杂的应用，底部导航栏能让用户更快地在“剧本”和“画布”之间切换。
